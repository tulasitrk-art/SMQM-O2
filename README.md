Project: Smart Mess & Queue Manager
Core Purpose: Real-time campus dining and queue management system to eliminate peak-hour mess congestion.
Real-Time Sync: Firebase Firestore real-time listeners (onSnapshot) for instant token updates and live wait times.
Concurrency Control: Strict 10-character roll number validation and status checks to prevent duplicate token bookings.
Dynamic Meal Scheduling: Admin controls to toggle meal windows (Breakfast, Lunch, Snacks, Dinner) in real time.
Flexible Menu Engine: Dual-mode menu system supporting both fixed weekly cycles and daily specials.
Mobile-First UI: Lightweight, responsive interface built with native JavaScript, HTML5, and CSS3.
Core Impact: Reduced physical wait times, eliminated hall overcrowding, and streamlined campus dining operations.
